# The Pitchside AI Development Workflow

These are the strict ground rules for all code assistance and development tasks on this project.

## 1. Code Generation Only
Write the requested code and modify the necessary files to achieve the stated goal. Do not preemptively write or execute any automated testing suites.

## 2. Stop for Manual Validation
Once the code for a specific feature or fix is written, stop all actions. Wait for manual validation. The user will test the changes locally in the browser to ensure the UI and logic behave correctly before proceeding.

## 3. Strict Version Control Ban
Under no circumstances are you permitted to stage, commit, merge, or push code to the repository. Do not run any Git commands autonomously. The user will explicitly request version control actions only when a feature is 100% validated and ready for deployment.

## 4. Maintain Brand Guidelines
When generating UI components or email templates, strictly adhere to the Pitchside aesthetic:
*   **Core Colors**: Black (#000000), White (#ffffff), and Neon Green (#cbff00).
*   **Typography**: Inter font.
*   **Vibe**: Modern, high contrast, and clean.

## 5. One Step at a Time
Do not anticipate the next feature and start building it without permission. Stick strictly to the current prompt to avoid bloating the codebase.
