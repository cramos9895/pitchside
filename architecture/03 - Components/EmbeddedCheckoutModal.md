# 🧩 EmbeddedCheckoutModal

**Type:** #component **Location:** `src/components/EmbeddedCheckoutModal.tsx`

## 📥 Props Received

- **isOpen** (boolean): Controls the modal's presence in the DOM.
- **onClose** (function): Callback used to dismiss the checkout session if the user chooses to cancel.
- **clientSecret** (string): The unique session token generated by the `/api/checkout` endpoint, required to initialize the Stripe provider.

## 🎛️ Local State & UI Logic

- **Vault-Focused Immersion**:
    - Uses an aggressive `backdrop-blur-md` and `bg-black/85` overlay to neutralize the background application. This "Visual Isolation" strategy is intended to maximize transaction focus and conversion.
- **Modern Stripe Embedding**:
    - Implements the newer `EmbeddedCheckoutProvider` pattern rather than the standard `Elements` API. This allows PitchSide to render Stripe’s hosted checkout experience _inside_ the app's DOM, providing a "Zero-Redirect" payment flow that supports higher complexity (discounts, Apple/Google Pay, and multi-tax calculations) while maintaining a native feel.
- **Live Connection Feedback**:
    - Features a pulsing `pitch-accent` LED indicator in the header, signaling to the user that they are in a "Active Secure Tunnel" with the payment processor.
- **Responsiveness & Sandboxing**:
    - The checkout iframe is wrapped in a `max-h-[90vh]` container with internal `overflow-y-auto`. This ensures that even if a user has multiple saved cards or long address forms, the modal remains perfectly centered and scrollable on mobile devices.
- **Tactile Dismissal**:
    - Standardizes the `zoom-in-95` entry animation and a high-contrast close button, allowing users to safely exit the payment funnel and return to the event discovery view without loss of state.

## ⚡ Actions & API Triggers

- **Stripe `EmbeddedCheckout` SDK**: The primary bridge to Stripe’s secure infrastructure.
- **`/api/checkout`**: The preceding backend trigger that creates the session and returns the `clientSecret`.
